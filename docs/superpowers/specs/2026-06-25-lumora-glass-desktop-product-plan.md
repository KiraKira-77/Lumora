# 光枢 Lumora Glass Desktop 产品方案

## 1. 市场判断

结论：有市场，但不是大众消费级大市场。更现实的是 Windows 效率用户、桌面混乱用户、设计/开发/办公重度用户的小众付费工具市场。

这个方向成立的原因：

- Windows 用户基数仍然足够大。
- Windows 原生搜索和桌面整理体验仍有明显痛点。
- 已有竞品证明需求存在：PowerToys Run、Flow Launcher、Listary、Everything、Stardock Fences。
- 这些竞品各自解决单点问题，但没有把“视觉启动器 + Dock 插槽 + 文件搜索 + 桌面整理”做成一个统一桌面体验。

主要风险：

- 单纯做启动器，会被 PowerToys Run、Flow Launcher、Listary 压住。
- 单纯做文件搜索，会被 Everything、Listary 压住。
- 单纯做桌面整理，会被 Fences 压住。
- 单纯做 Mac 风 UI，会变成皮肤工具，缺少长期价值。

所以 Lumora 不能只做一个漂亮启动器。它必须做成：

> Windows 上的可视化快捷桌面。

## 2. 产品定位

### 2.1 一句话定位

光枢 Lumora 是一个 Mac 风格的 Windows 快捷桌面，用户可以拖入常用应用、文件、文件夹和网页，通过 Dock、快捷键和搜索快速打开，并自动整理桌面文件。

### 2.2 用户心智

- PowerToys Run：键盘启动。
- Everything：极速文件搜索。
- Fences：桌面图标整理。
- 浮光：可视化快捷键面板。
- Lumora：把 Windows 桌面改造成可拖拽、可搜索、可收纳的快捷工作台。

### 2.3 核心价值

用户打开 Lumora 后应该立刻感受到：

- 桌面可以清空。
- 常用应用可以放到底部插槽。
- 应用、文件、文件夹、网页都能快速打开。
- 文件能快速搜到。
- 桌面文件能一键分类收纳。
- 整体体验比 Windows 原生更顺、更漂亮。

## 3. 目标用户

### 3.1 核心用户

Windows 重度桌面用户。

典型特征：

- 桌面图标多。
- 经常找文件。
- 经常打开固定应用和文件夹。
- 对 Windows 默认开始菜单、搜索、桌面整理不满意。
- 喜欢 Mac 风格的 Dock、玻璃质感和顺滑动效。
- 愿意花时间配置一个顺手的效率工具。

### 3.2 首批用户场景

#### 办公用户

- 常用微信、钉钉、浏览器、Office、项目文件夹。
- 桌面堆满临时文件。
- 需要快速收纳桌面。

#### 设计/内容用户

- 常用设计软件、素材文件夹、截图、下载目录。
- 桌面经常堆图片、PSD、PDF、压缩包。
- 需要快速分类和拖拽入口。

#### 程序员/技术用户

- 常用 IDE、终端、浏览器、项目目录、文档站。
- 需要快速启动常用项目和文件夹。
- 对快捷键和启动速度敏感。

## 4. 产品主线

V1 主线：

> 拖入常用项 → 绑定快捷入口 → 快速打开 → 搜索文件 → 收纳桌面。

V1 不再主推 Workspace。Workspace 可以以后演化为“场景组”，但不是第一版心智。

## 5. V1 核心模块

### 5.1 Glass Launcher

中央玻璃启动器。

功能：

- 快捷键呼出，默认 Alt + Space。
- 搜索应用、文件、文件夹、网页、命令。
- 默认展示快捷键矩阵。
- 支持拖入应用、文件、文件夹、网址。
- 每个快捷项可以绑定按键。
- 支持最近打开。
- 支持固定常用项。

关键体验：

- 呼出必须快。
- 输入必须无卡顿。
- 动画必须短，150 到 220ms。
- 默认界面必须一眼能懂。

### 5.2 Bottom Dock 插槽

屏幕中下方的快捷插槽。

功能：

- 可常驻。
- 可自动隐藏。
- 可仅桌面显示。
- 支持拖入应用、文件夹、文件、网页。
- 支持拖拽排序。
- 支持分组。
- 支持右键菜单。
- 支持从启动器搜索结果固定到 Dock。

默认插槽：

- 文件搜索
- 桌面收纳
- 下载目录
- 浏览器
- 截图
- 设置

用户价值：

- 用户可以移除 Windows 桌面图标。
- 常用入口集中到底部 Dock。
- Windows 桌面变干净。

### 5.3 File Search 文件搜索

文件搜索不建议自研索引。

V1 应集成 Everything 或调用 Everything SDK/HTTP/IPC 能力。Lumora 负责 UI、排序、动作和入口。

功能：

- 搜文件名。
- 搜文件夹。
- 支持模糊搜索。
- 支持按类型筛选。
- 支持打开文件。
- 支持打开所在目录。
- 支持复制路径。
- 支持固定到 Dock。
- 支持移动到分类文件夹。

不做：

- 不做全文内容索引。
- 不做云盘搜索。
- 不做复杂高级查询 UI。

### 5.4 Desktop Organizer 桌面整理

这是 Lumora 的强差异点之一。

功能：

- 扫描桌面文件。
- 按类型生成分类预览。
- 用户确认后移动文件。
- 支持撤销整理。
- 支持桌面收纳箱。
- 支持规则化分类。

默认分类：

- Images
- Docs
- Archives
- Installers
- Videos
- Projects
- Inbox

关键原则：

- 不静默移动文件。
- 整理前必须预览。
- 整理后必须可撤销。
- 保留日志。

### 5.5 Widgets 小组件

V1 小组件必须克制。

只做：

- 日历。
- 待办。
- 最近文件。
- 剪贴板最近 5 条。
- 桌面收纳箱。

不做：

- 天气。
- 新闻。
- 壁纸社区。
- AI 对话主界面。
- 插件市场。

## 6. UI 风格

方向：

> Apple Liquid Glass inspired Windows utility UI.

设计原则：

- 中央启动器使用玻璃材质。
- Dock 使用半透明磨砂底座。
- 控件轻微高光。
- 图标圆角方形。
- 背景可以采样用户壁纸。
- 内容区域保持可读，不全屏毛玻璃。
- 动效快速、轻、顺。

不要做：

- 大面积紫蓝渐变。
- 过度发光。
- 过度拟物。
- 伪装成 macOS 三色窗口按钮。
- 整页毛玻璃导致可读性下降。

## 7. 首次使用流程

1. 用户安装 Lumora。
2. 首次启动显示底部 Dock 和中央启动器。
3. 引导用户拖入第一个应用。
4. 用户把桌面图标、开始菜单应用、文件夹拖入 Dock。
5. Lumora 自动生成图标和快捷键。
6. 用户按 Alt + Space 呼出启动器。
7. 用户通过搜索或按键打开应用。
8. Lumora 提示是否扫描桌面文件。
9. 展示分类预览。
10. 用户确认后，桌面文件移动到 Lumora 分类文件夹。
11. 用户可以隐藏 Windows 桌面图标。

## 8. 商业模式

### 免费版

目标：让用户体验核心价值。

包含：

- 启动器。
- 最多 12 个 Dock 插槽。
- 基础文件搜索。
- 每周 3 次桌面整理。
- 基础快捷键矩阵。

### Pro

建议一次性买断或订阅都可测试。

包含：

- 无限 Dock 插槽。
- 多 Dock 分组。
- 高级快捷键矩阵。
- 自动桌面整理规则。
- 整理历史和撤销增强。
- 小组件。
- 搜索结果高级动作。
- 配置导入导出。
- 主题和玻璃强度调节。

### 企业版

暂不设计。

## 9. MVP 范围

第一版只做：

- Alt + Space 启动器。
- 快捷键矩阵。
- 拖入应用/文件/文件夹/网页。
- 底部 Dock 插槽。
- Dock 自动隐藏。
- 文件搜索。
- 桌面整理预览。
- 一键分类。
- 撤销整理。
- 基础设置。

第一版不做：

- Workspace。
- 云同步。
- 插件市场。
- AI 助手。
- 复杂剪贴板。
- 复杂截图编辑。
- 多设备同步。
- 团队功能。
- 桌面美化社区。

## 10. 成功指标

核心指标：

- 用户是否隐藏或清空了原 Windows 桌面图标。
- 用户每天通过 Lumora 打开应用/文件的次数。
- 用户是否持续使用 Dock。
- 用户是否使用桌面整理。

具体指标：

- D1 留存。
- D7 留存。
- 每日启动器呼出次数。
- 每日 Dock 点击次数。
- 用户平均固定项数量。
- 桌面整理使用次数。
- 桌面整理撤销率。
- 搜索成功打开率。
- Pro 转化率。

## 11. 竞争策略

不和 Everything 比索引速度。

不和 PowerToys Run 比系统工具数量。

不和 Fences 比复杂桌面容器。

不和 Flow Launcher 比插件生态。

Lumora 的差异化应该是：

> 更好看、更直观、更适合普通 Windows 用户清理桌面和建立快捷入口。

## 12. 最大风险

### 风险 1：变成皮肤工具

只做 Mac 风 UI 没价值。

处理：

必须绑定实际效率收益：快速打开、快速搜索、快速整理。

### 风险 2：文件整理引发用户不信任

移动文件是高风险动作。

处理：

必须预览、确认、撤销、记录日志。

### 风险 3：启动器竞争激烈

启动器本身不是稀缺品。

处理：

用 Dock 和桌面整理建立差异化，不只做搜索框。

### 风险 4：性能不够顺

这种产品如果卡，直接失败。

处理：

V1 把性能列为核心需求，不堆功能。

## 13. 最终结论

Lumora 有市场，但必须避开“又一个启动器”的陷阱。

最现实的产品方向是：

> Mac 风格的 Windows 快捷桌面：底部 Dock 插槽 + 玻璃启动器 + 文件搜索 + 桌面整理。

这个方向的第一卖点不是专业工作流，而是用户能立刻看懂的收益：

- 桌面变干净。
- 常用入口更顺手。
- 文件更快找到。
- Windows 更像一个高级、可控、漂亮的桌面系统。

如果 V1 能把“拖入、打开、搜索、整理”四件事做得足够顺，就有机会形成小众但真实的付费产品。
